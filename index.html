El camino del desarrollo

Las cuatro P de la Ingeniería del Software

Para iniciar, recordemos a que se hace referencia cuando hablamos de las cuatro P, estas son unas características a tener en consideración cuando se desarrolla un software y se desea una gestión adecuada, eficaz y eficiente. Para nuestro tema de fases de proyecto, dicha definición toma relevancia al "proyecto" ser una de las cuatro P.

Si deseas profundizar el tema de las cuatro P de la Ingeniería del Software, puedes acceder a un mapa mental sobre el tema dando clic al botón.

P de proceso

Esta característica hace énfasis en las actividades a efectuar para el desarrollo del proyecto o para la creación del producto, a dichas actividades se le denominan fases y estas se pueden presentar de dos maneras:

Corta:

  1. Requerimiento
  2. Diseño
  3. Implementación
  4. Pruebas
  5. Mantenimiento

Larga:

  1. Iniciación
  2. Planeación
  3. Análisis
  4. Diseño
  5. Implementación
  6. Pruebas
  7. Mantenimiento

Entendamos cada fase

Para su explicación nos basaremos en la versión larga

1. Iniciación
  • Se plantea del software por una necesidad o idea innovadora.
  • Al haber clientes deben conocerse sus necesidades y percepciones.
  • Es posible realizar un análisis del mercado para conocer desarrollos similares, también puede preverse la aceptación del público.
2. Planeación
  • Se gestionan temas como el tiempo, los costos, recursos humanos y demás.
  • Planteamiento de un plan de gestión de configuración para supervisión de cambios y versiones en el software.
  • Planteamiento de un plan de aseguramiento de calidad en pro a la calidad del software.
3. Análisis
  • Se toman los requerimientos que exprese el cliente, recordemos que estos especifican funcionalidades para el software. Cabe mencionar que a este proceso se le conoce como ingeniería de requerimientos.
  • Recopilación de información útil.
  • Inicio del diseño por diagramas de casos de uso.
4. Diseño
  • Para este puntos los requerimientos expuestos por el cliente y los encontrados ya deben estar definidos.
  • Implementación del diagrama de secuencias y de clases. Es recomendable que el diagrama mencionado en la fase anterior esté culminado.
  • Se precisa como será la arquitectura general del sistema.
5. Implementación
  • Todas las indicaciones de la fase de diseño deben estar completas para iniciar esta fase, pues en base a los resultados obtenidos, se inicia la codificación o programación del software.
  • Es posible apoyarse de tecnologías ya existentes para dicha codificación, como frameworks o librerías.
6. Pruebas
  • Se revisa la calidad del software realizado en la fase anterior.
  • ¿Los requerimientos dados desde el inicio por el cliente están plasmados en el software?
  • Para garantizar funcionalidades, es recomendable realizar pruebas con los usuarios finales pensados para la utilización del software.
7. Mantenimiento
  • Para este entonces el software ya fue entregado.
  • Se hacen reparaciones si se requieren como forma de soporte al cliente.
  • Pueden generarse nuevos requerimientos.
  • Cada modificación o mantenimiento produce una nueva versión del software.

¿Qué podemos concluir?

Las fases de proceso toman relevancia en la Ingeniería de Software por hacer el desarrollo mas facil y organizado para todos, pues como se denotó en la información dada, estas son útiles cuando se plantea el inicio de un proyecto. Mas allá de que tengamos los conocimientos técnicos requeridos para dicho proyecto, se es importante tener un plan de seguimiento para evitar problemas con la organización y asi evitar perdidas en tiempo, dinero y confiabilidad por parte del cliente.
